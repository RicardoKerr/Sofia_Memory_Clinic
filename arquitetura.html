Estrutura de Roteamento e Serviços

1. Frontend Modular SPA

O cliente da Sofia é estruturado em uma Single Page Application leve escrita em Javascript Modular (ES6). As telas são renderizadas de forma reativa a partir de classes no diretório components/. A controller app.js sincroniza os dados do paciente ativo e injeta as visões correspondentes sem recarregar o navegador.

2. Proxy de API Desacoplado

Toda a comunicação externa da aplicação passa por proxy em services/api.js. Na demonstração atual, os dados são mockados no LocalStorage. Em produção, basta mapear as chamadas REST para um Client SDK do Supabase, evitando vazamento de lógica nos componentes de visualização.

// Exemplo de chaveamento futuro async getEvents(profileId) { // Retorna do Supabase de forma nativa const { data } = await supabase .from('events') .select('*') .eq('profile_id', profileId); return data; }

3. Pipeline de Conversação de Voz

O processamento de áudio utiliza a Web Speech API local para transcrição de fala imediata. Quando configurada a chave de API do ElevenLabs, a síntese de voz (conversão de texto em áudio respondido) é redirecionada para a nuvem da ElevenLabs, gerando uma experiência de fala realista e natural.

4. OCR e Visão Computacional

O fluxo de câmera acessa o hardware via navigator.mediaDevices e captura frames em tempo real via canvas 2D. O processador está estruturado para redirecionar arquivos base64 para pipelines de processamento da OpenAI (GPT-4o Vision), retornando dados tabulados de receitas e laudos clínicos automaticamente.
